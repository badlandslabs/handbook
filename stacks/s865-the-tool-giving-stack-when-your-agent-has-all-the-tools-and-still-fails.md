# S-865 · The Tool-Giving Stack — When Your Agent Has All the Tools and Still Fails

You gave your agent a browser, a code executor, a database, and 12 MCP servers. It still fails. The problem is rarely the model — it's the gap between "tools available" and "tools the agent can actually use correctly." Tool use is not a solved problem just because the function-calling API exists.

## Forces

- **The token tax on tool lists.** Loading all tool definitions upfront is the default but it's ruinously expensive at scale. A 5-server MCP setup with 58 total tools consumes ~55K tokens before the first user message. Anthropic has seen setups hit 134K tokens just for tool definitions. — [Anthropic Engineering: Advanced Tool Use](https://www.anthropic.com/engineering/advanced-tool-use)
- **Stale state is the real browser-agent problem.** The dominant failure mode in browser automation is not model misunderstanding — it's the agent reasoning from a browser state that's out of sync with what actually happened. The page rendered differently than expected, a click triggered an async update, a dialog appeared the model didn't anticipate. — [HN: Agent Browser Protocol (ABP)](https://news.ycombinator.com/item?id=47336171)
- **MCP's promise vs. its blast radius.** Model Context Protocol solved the integration plumbing — "USB-C for AI" — but it shifted the problem to tool selection, privilege scoping, and prompt injection. Giving an agent 40 Windows tools (mouse, keyboard, registry, services) is powerful until one of those tools does something unintended. — [r/LocalLLaMA: WinRemote MCP](https://www.reddit.com/r/LocalLLaMA/comments/1r1t0dt/i_built_an_mcp_server_that_gives_ai_agents_full/)
- **Computer use when an API exists is a smell.** Anthropic's own documentation warns: if you're automating the UI of something that exposes a machine interface, you're paying 466–499 extra system-prompt tokens plus ~735 tool-definition tokens per call for screenshot overhead that adds nothing. — [LaoZhang AI: Claude Computer Use](https://blog.laozhang.ai/en/posts/claude-computer-use)

## The Move

The core discipline is **tool curation over tool accumulation** — giving the agent the right tools, scoped correctly, discovered on demand rather than loaded wholesale:

- **Scope MCP servers to least-privilege paths.** Don't give the filesystem MCP access to `$HOME`. Give it `/tmp/project-name` only. The MCP ecosystem defaults to broad access; production agents need surgical boundaries. — [OpenCode Guide: MCP Use Cases](https://opencodeguide.com/en/mcp)
- **Prefer API tools over UI automation.** If a machine interface (REST API, CLI) exists, use it. Computer use through screenshots is for systems with no API — accepting the token cost and state-synchronization risk as the price of last resort. — [Anthropic Engineering: Advanced Tool Use](https://www.anthropic.com/engineering/advanced-tool-use)
- **Load tools on-demand, not at startup.** Anthropic's Tool Search Tool pattern: the agent searches for relevant tools when it encounters a task that requires one, rather than holding all 58 tool definitions in context. This cuts token cost dramatically for large tool inventories. — [Anthropic Engineering: Advanced Tool Use](https://www.anthropic.com/engineering/advanced-tool-use)
- **Use sandboxed code execution instead of shell access.** The OpenAI Agents SDK's `CodeInterpreterTool` runs in an auto-managed container. This isolates the agent from the host system while still enabling computation. — [OpenAI Agents SDK: Tools](https://openai.github.io/openai-agents-js/guides/tools)
- **Pair tool definitions with usage examples.** Anthropic's Tool Use Examples feature lets you embed correct invocation patterns alongside tool definitions — reducing miscalls more than verbose descriptions do. — [Anthropic Engineering: Advanced Tool Use](https://www.anthropic.com/engineering/advanced-tool-use)
- **Freeze state before each browser action.** ABP's approach: after every agent action (click, type), freeze JavaScript execution and rendering, then capture the resulting state before returning control. This closes the stale-state gap that causes most browser-agent failures. — [HN: Agent Browser Protocol (ABP)](https://news.ycombinator.com/item?id=47336171)

## Gotchas

- **Adding a browser tool doesn't make UI automation safe.** Anthropic recommends dedicated VMs with minimal privileges and limited internet access for computer use. Prompt injection through web pages is a real attack surface — a page the agent visits can influence its behavior. — [LaoZhang AI: Claude Computer Use](https://blog.laozhang.ai/en/posts/claude-computer-use)
- **Tool token cost is invisible until it isn't.** A team that naively wires up 5 MCP servers can be spending more tokens on tool definitions than on the actual conversation. Profile this before shipping, not after the bill arrives.
- **More tools → more mis-selection.** Agents with large tool inventories pick the wrong tool more often than agents with narrow, well-described tool sets. Curation beats coverage.
- **Tool outputs can hallucinate.** A database query tool might return incorrect rows; a search tool might return irrelevant results. The agent then acts on false information. Treat tool outputs as untrusted input, not ground truth. — [GitHub: awesome-agent-failures](https://github.com/vectara/awesome-agent-failures)
