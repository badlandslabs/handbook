# [S-1028] · Synthetic Trajectory Degeneration

You fine-tuned your agent on its best production trajectories. Six weeks and three retraining cycles later, it handles the known cases beautifully — and fails on anything slightly new. It has become a specialist in your training distribution, and nothing else. The model hasn't degraded in the traditional sense. It has *converged* — into a narrower, more brittle version of itself. This is synthetic trajectory degeneration, and it is the silent failure mode of every team that builds a self-improving agent without anchoring to ground truth.

## Forces

- **Recursive self-training is a compression algorithm.** Every fine-tuning cycle pulls the model toward the distribution of its own outputs. With enough cycles, the model can only reproduce what it has already seen. Novel inputs — the long tail that matters in production — fall outside the compressed distribution.
- **Trajectory quality decays under pressure.** Early synthetic trajectories are generated by a capable frontier model. After each fine-tuning cycle, the agent generating trajectories is slightly less capable on edge cases. Later cycles are trained on degraded data. The spiral is self-reinforcing.
- **Successful trajectories are survivorship-biased.** You collect what worked and train on it. But "worked" means "worked on the training distribution." The failures — which contain the signal about where the agent is wrong — are discarded. Each cycle amplifies the survivorship bias.
- **Eval benchmarks mask the narrowing.** Your eval score may stay flat or improve because the benchmark overlaps with the training distribution. The capability loss is in the *complement* — the cases not in the benchmark, not visible until they hit production.
- **Foundation models forget, fine-tuned models never recover.** A base frontier model retains broad capability. A fine-tuned agent that has degenerated cannot be recovered by prompting alone — the knowledge was overwritten in the weights, not suppressed.

## The Move

**1. Anchor to a ground-truth anchor set.**
Maintain a fixed, human-curated validation set (minimum 500 cases) that represents the full operational domain — not just the training cases. This set is never used for training and is re-evaluated after every fine-tuning cycle. If the anchor score drops, the cycle is rejected regardless of training-set improvement.

**2. Cap recursive fine-tuning cycles.**
After 2–3 cycles without a >5% improvement on the anchor set, stop. The marginal gain from continued self-training does not offset the degeneration risk. Invest in human-in-the-loop trajectory collection instead.

**3. Inject diversity at every cycle.**
Mix 30–40% of your anchor set into each training batch alongside synthetic trajectories. This re-anchors the model to the full distribution rather than the synthetic subset. Without this, each cycle is a further compression.

**4. Track distribution overlap.**
After each fine-tuning cycle, measure the KL divergence between the fine-tuned model's output distribution and the base frontier model on held-out prompts. A KL divergence > 0.3 on task-adjacent prompts is an early warning signal of narrowing. Degrade gracefully: reduce the proportion of synthetic data in the next cycle.

**5. Maintain a synthetic-free baseline.**
Keep one deployment track running the base (unfine-tuned) frontier model. Compare its performance on anchor cases against the fine-tuned version. If the base model outperforms on >10% of anchor cases, the fine-tuned agent has degenerated and should be replaced or re-anchored.

```python
# Minimal trajectory degeneration monitor
import numpy as np
from scipy.stats import entropy

def measure_degeneration(baseline_logits, fine_tuned_logits, anchor_prompts):
    """
    baseline_logits: logits from base frontier model, shape (N, V)
    fine_tuned_logits: logits from fine-tuned agent, shape (N, V)
    Returns KL divergence and degeneration score.
    """
    base_probs = softmax(baseline_logits, axis=-1)
    ft_probs = softmax(fine_tuned_logits, axis=-1)

    # KL divergence on anchor set (higher = more divergent = possible narrowing)
    kl_divs = [
        entropy(base_probs[i] + 1e-8, ft_probs[i] + 1e-8)
        for i in range(len(anchor_prompts))
    ]
    mean_kl = np.mean(kl_divs)

    # Degeneration score: fraction of anchor cases where base outperforms FT
    base_wins = np.mean(np.argmax(base_probs, axis=-1) != np.argmax(ft_probs, axis=-1))

    return {
        "mean_kl_divergence": mean_kl,
        "base_model_wins_pct": base_wins,
        "degenerate": mean_kl > 0.3 or base_wins > 0.10
    }

def should_continue_training(degeneration_score, cycle_count, anchor_improvement):
    if cycle_count >= 3:
        return False
    if degeneration_score["degenerate"]:
        return False
    if anchor_improvement < 0.05:
        return False
    return True
```
