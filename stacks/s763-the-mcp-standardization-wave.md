# S-763 · The MCP Standardization Wave

In under a year, the Model Context Protocol went from Anthropic's internal idea to the USB-C of AI tooling — a vendor-neutral standard that every major AI player adopted, hundreds of Fortune 500 companies deployed, and the Linux Foundation now governs. Teams building agentic stacks that ignore it are reinventing a wheel that's already standardized.

## Forces

- **Custom tool integrations rot fast.** Every agent system starts by hand-rolling HTTP tool definitions. As tools multiply and models evolve, these integrations drift out of sync — schema mismatches, missing auth propagation, and silent failures accumulate.
- **Sandboxing and security are now non-negotiable.** Agents that can call tools, execute code, and reach external APIs need isolation layers. The agent stack is stratifying around this concern — execution sandboxes (E2B, Modal, Firecracker) are becoming a distinct, separately maintained layer.
- **Multi-vendor model usage is now the norm.** 37% of enterprises use five or more AI models in production. Each model has its own tool-calling schema. Without a shared protocol, every (model, tool) pair is a custom integration project.
- **The evaluation gap is real.** 89% of agent teams have observability but only 52% have structured evals. Tool-call accuracy degrades below 90% once you exceed 12 tools — and without standardized interfaces, you can't measure when it happens.

## The move

**Build your tool layer on MCP, not custom schemas.**

- **Adopt MCP as your integration contract.** MCP uses JSON-RPC 2.0 over stdio or HTTP, with SDKs in Python, TypeScript, Java, and C#. Define tools once as MCP servers and any MCP-compatible client (Claude, GPT, Gemini) can consume them without per-model re-implementation. MCP server downloads grew from ~100K in November 2024 to 97M+ monthly by late 2025, with 5,800+ servers and 300+ clients published.
- **Layer your sandboxing separately from orchestration.** Philipp Dubach's analysis identifies the agent stack stratifying into six distinct layers (model, orchestration, tool, sandbox, memory, observability), each with different defensibility profiles and change rates. Sandboxing (E2B, Modal, Firecracker wrappers like Shuru) should be its own maintained layer — not baked into the orchestrator.
- **Use typed handoffs in multi-agent workflows.** Untyped handoffs kill multi-agent workflows faster than any other issue. When agent A passes context to agent B, define the expected shape of that message schema explicitly. This is where LangGraph's typed state edges provide an advantage over CrewAI's implicit delegation.
- **Instrument at the tool-call level, not just the LLM level.** Amazon's agent evaluation framework emphasizes tracking accuracy of tool selection decisions, not just output quality. A tool-call failure rate of 10% in a 20-step workflow means the task fails 90% of the time — yet most teams only monitor LLM token counts.
- **Evaluate multi-agent coordination with HITL, not just automated metrics.** With multiple agents in play, emergent behaviors appear that automated evals miss: coordination failures, contradictory agent recommendations, and logical inconsistency across agent contributions. Amazon's experience shows human-in-the-loop evaluation catches these at rates automated metrics cannot match.
- **Apply cost controls at the orchestration layer, not the model layer.** Multi-agent inference costs $5–8 per complex task in 4-agent workflows (RaftLabs, 2025). Supervisor pattern enables centralized cost budgeting: the supervisor agent decides which sub-agents to invoke and can enforce per-task token budgets. Peer/market patterns let costs diffuse across agents and become harder to audit.

## Gotchas

- **MCP's base protocol lacks production primitives.** Identity propagation, tool budgeting, and structured error semantics aren't in the base spec — they need broker pipelines (CABP), adaptive timeouts (ATBA), and actionable error taxonomies (SERF) layered on top.
- **Code-execution MCP models (CE-MCP) trade security for efficiency.** Context-decoupled execution (agent emits a program, sandbox runs it) reduces token usage 70%, turns 83%, and latency 3× — but requires deep sandboxing and semantic validation of execution plans since the attack surface expands significantly.
- **Multi-agent specialization only pays off past a complexity threshold.** If your workflow is 3–5 steps with 3–5 tools, a single agent is faster, cheaper, and easier to debug. The specialization tax (typed handoffs, coordination overhead, more inference calls) only recovers its cost when genuine boundaries exist.
- **LangGraph vs CrewAI tradeoff is about rewrites, not features.** LangGraph's graph-based state management has a steeper learning curve but prevents painful rewrites 6–12 months in. CrewAI ships prototypes faster but teams hit scalability walls within 6–12 months of real production load.
