# S-1322 · The Orchestration Decision Stack — When You're Chaining Agents and Every Framework Tells You This Is Trivial

Your task has outgrown a single LLM call. You need multiple agents, shared state, tool pipelines, and failure recovery. Every framework hand-waves the hard part: the decision between "I'll use LangGraph," "I'll use CrewAI," or "I'll build my own." That decision has lasting consequences for debugging, cost, and how far you can scale.

## Forces

- **Flexibility vs. boilerplate** — Rigid role-based pipelines (CrewAI) are fast to set up but constrain workflow topology. Graph-based (LangGraph) gives you full control but requires explicit state management.
- **Production readiness vs. iteration speed** — Rolling your own orchestration gives maximum flexibility but ships you a custom observability problem and failure-handling gap.
- **Cost optimization vs. quality** — Routing tasks to cheap models (Haiku-class) for classification and expensive models (Opus-class) for planning is the right idea, but routing logic is itself a non-trivial problem.
- **Centralized control vs. emergent coordination** — A supervisor/hierarchical model gives you auditability and deterministic execution; mesh-based coordination (A2A/MCP) enables agents to self-organize but is harder to reason about.
- **Enterprise governance vs. developer experience** — Every layer of policy, approval, and tracing you add in production makes the orchestration heavier.

## The Move

The pattern that emerges from production multi-agent systems in 2025–2026 is a **three-tier hierarchical dispatch model**, combined with protocol-based inter-agent communication (A2A + MCP). The exact framework choice matters less than getting the architecture right:

- **The Orchestrator (Planning Tier)** — Runs the most capable model (Opus-class or equivalent). Handles ambiguous requirements, plans execution, resolves conflicts between specialist outputs. This is the most expensive tier and the most consequential: bad planning cascades downstream.
- **The Router (Dispatch Tier)** — A fast, cheap model (Haiku-class or GPT-4o-mini equivalent) classifies each incoming task by type and complexity, then routes to the appropriate specialist. Routing decisions happen in milliseconds and consume minimal tokens. Some architectures embed routing logic directly in the orchestrator; others use a dedicated lightweight routing agent for cleaner separation.
- **The Specialists (Execution Tier)** — Purpose-built agents for specific domains: code generation, data retrieval, summarization, API calls. Each runs on the cheapest model that can reliably handle its domain. A summarization agent might run on Haiku-class; a code-review agent needs Opus-class.
- **Communication protocol** — MCP (Model Context Protocol) handles tool calling and resource access. A2A (Agent-to-Agent) handles inter-agent state sharing and task delegation. These two protocols are now de facto standards: MCP gives agents their tools; A2A gives agents their language.
- **Process type selection** — Within CrewAI (if used), the choice between sequential and hierarchical process types is the most common source of production incidents. Sequential is right when task order is fixed and failure should halt the pipeline. Hierarchical is right when a manager agent should coordinate sub-agents and synthesize results — but it adds latency and a new failure point.
- **Observability as a first-class concern** — Production teams instrument traces from day one. LangSmith, Wayfound, and custom abstractions all appear in production stacks. Without traces, debugging a multi-agent pipeline is archaeology.
- **Human-in-the-loop checkpoints** — LangGraph's `interrupt_before`/`interrupt_after` breakpoints let you pause a running graph for approval. In production this is non-negotiable for high-stakes actions (writes, deletions, external API calls).

## Gotchas

- **Don't pick a framework before choosing your topology.** LangGraph is a state machine. CrewAI is a role-based team. They solve different problems. Define your agent interaction pattern first; the framework follows.
- **Sequential vs. hierarchical is a production-critical decision in CrewAI.** Sequential halts on failure. Hierarchical adds a manager bottleneck. Switching after deployment is painful.
- **Routing logic is a model-dependent fragile thing.** If your router is a prompt-based classifier, it drifts with model updates. Treat routing as a first-class component, not a prompt afterthought.
- **Mesh/emergent coordination (A2A-only) is powerful but opaque.** Agents that self-organize without a central orchestrator are harder to debug and audit. For regulated environments, a supervisor/hierarchical model is safer.
- **Checkpoint everything.** Multi-agent pipelines with no resumability strategy will re-execute the entire workflow on restart. LangGraph's `interrupt_before`/`interrupt_after` breakpoints are one solution; explicit state serialization is another.
